# 🐝 Java Bee Simulations 🐝

A university team project (3 contributors) built around bee–flower ecosystem simulations and bee-inspired optimization algorithms in Java. The work was evenly split across the team and documented through annotations and comments.

The repository contains multiple simulation variants and demonstrates a broad range of software engineering skills, including:

- **Object-oriented programming**: inheritance, polymorphism, interfaces, abstract classes, encapsulation
- **Functional programming**: lambdas, method references, streams, higher-order functions
- **Procedural programming**: stepwise simulation logic and algorithmic control flow
- **Parallel programming**: concurrent batch execution using `ExecutorService`, task submission, and result aggregation
- **Generic programming and abstraction**: reusable data structures and algorithm interfaces
- **Design by Contract**: explicit preconditions, postconditions, invariants, and history constraints
- **Design patterns**: Visitor, Factory Method, Abstract Factory, and other object-oriented design techniques
- **Algorithmic design**: heuristic / population-based search inspired by bee foraging behavior

## What the project covers

The project is split into several simulation modules, each focusing on a different software design challenge:

- **Sim1**: parallel execution of simulation batches and statistical result collection using factory-based simulation setup and streams
- **Sim2**: modeling bee species and behaviors with inheritance and polymorphism
- **Sim3**: generic ordered set abstractions backed by graph-based relations
- **Sim4**: contract-driven simulation architecture using formal annotations and the Visitor pattern
- **Sim5**: bee-inspired optimization algorithm implemented with functional pipelines and stream processing
